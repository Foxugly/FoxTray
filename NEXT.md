## 2. GitHub Actions CI

The workflow at `.github/workflows/test.yml` runs `pytest` on `windows-latest` + Python 3.14.

**Check the first run:** https://github.com/Foxugly/FoxTray/actions

**Likely first-run issue:** `actions/setup-python@v5` may not yet offer Python 3.14 as a prebuilt — in that case the job will either download the installer (slower but works) or fail. If it fails, edit `.github/workflows/test.yml`:

```yaml
      - name: Set up Python 3.14
        uses: actions/setup-python@v5
        with:
          python-version: "3.14"
          allow-prereleases: true   # add this line
```

Or downgrade to `"3.13"` if 3.14 isn't ready yet — the codebase uses no 3.14-exclusive features.

## 3. Building the standalone `FoxTray.exe`

One-time install:

```powershell
./.venv/Scripts/python.exe -m pip install pyinstaller>=6.0
```

Build (whenever `.py`, assets, or `__version__` change):

```powershell
./.venv/Scripts/python.exe scripts/gen_icons.py
./.venv/Scripts/python.exe -m PyInstaller --clean --noconfirm foxtray.spec
```

Output: `dist/FoxTray.exe` (~25-35 MB). Distribute by copying that file + a `config.yaml` to a folder of your choice.

See `docs/packaging.md` for the reference.

## 4. Replacing the tray icons

### What currently exists

- `assets/icon_running.png` — green disc (project healthy + url_ok)
- `assets/icon_partial.png` — orange disc (one component down, or starting)
- `assets/icon_stopped.png` — grey disc (no active project)
- `assets/foxtray.ico` — multi-resolution Windows icon for `FoxTray.exe` (derived from `icon_running.png`)

All four are auto-generated by `scripts/gen_icons.py`. Running the script overwrites them with the default coloured discs.

### Specs the files must meet

**Tray icons (3 PNGs):**

| Property | Value |
|---|---|
| Format | PNG (with transparency) |
| Mode | RGBA (4 channels — R, G, B, alpha) |
| Recommended size | 32×32 pixels (what pystray asks for) |
| Maximum size | 64×64 acceptable; Pillow downscales automatically |
| Filename | `icon_running.png`, `icon_partial.png`, `icon_stopped.png` (exact) |
| Location | `assets/` at repo root |

Pystray's `Icon` class passes the `PIL.Image` directly to the Win32 tray API. A fully transparent background around the coloured shape is important — otherwise you get a square with a colour behind the shape.

**Exe icon (`foxtray.ico`):**

| Property | Value |
|---|---|
| Format | ICO (multi-resolution container) |
| Embedded sizes | 16×16, 32×32, 48×48, 256×256 |
| Mode | RGBA |
| Location | `assets/foxtray.ico` |
| Used by | PyInstaller via `icon='assets/foxtray.ico'` in `foxtray.spec`; Windows Explorer shows the 256×256 version when the file is big, smaller sizes when it's compact |

### Replacing the icons

You have three options. Pick the one that fits your workflow.

#### Option A — drop-in replacement of the 3 PNGs (simplest)

1. Create your three custom icons in any editor (Affinity, Figma, Krita, …).
2. Export each as a 32×32 RGBA PNG (transparent background required).
3. Overwrite the files in `assets/` keeping the exact same filenames.
4. Regenerate the `.ico` (optional — only if you want the exe icon to follow):

```powershell
./.venv/Scripts/python.exe -c "from PIL import Image; Image.open('assets/icon_running.png').resize((256, 256), Image.Resampling.LANCZOS).save('assets/foxtray.ico', format='ICO', sizes=[(16,16),(32,32),(48,48),(256,256)])"
```

5. Reload the tray: close and relaunch `python main.py tray` (or rebuild the `.exe`).
6. Commit `assets/icon_*.png` and `assets/foxtray.ico`.

**Caveat:** as soon as someone runs `scripts/gen_icons.py` again, your custom PNGs are overwritten by the default coloured discs. If you want them preserved, use Option B or C.

#### Option B — let the generator produce them from larger source images (reproducible)

1. Create your three high-resolution source images (e.g., 256×256 or SVG exported to PNG).
2. Save them anywhere — conventional location: `assets/source/icon_running_src.png` etc.
3. Edit `scripts/gen_icons.py`:
   - Replace the `draw.ellipse(...)` block with `img = Image.open(source_path).convert('RGBA')`.
   - Resize to 32×32 with `img = img.resize((32, 32), Image.Resampling.LANCZOS)`.
4. Run `./.venv/Scripts/python.exe scripts/gen_icons.py` — the three `icon_*.png` files and `foxtray.ico` are regenerated from your sources.
5. Commit the sources + the generated outputs.

#### Option C — commit custom PNGs + remove the generator call for them (keeps ICO auto-generated)

1. Drop in your custom PNGs as in Option A.
2. Edit `scripts/gen_icons.py` — delete (or comment out) the `draw.ellipse(...)` PNG loop. Keep only the `_write_ico()` call at the end.
3. Now `gen_icons.py` is only responsible for the `.ico` (auto-derived from your custom `icon_running.png`), and your PNGs stay untouched.
4. Commit the three PNGs + the edited script.

### After regenerating: rebuild the .exe

The `.exe` bundles the PNGs at build time (they live inside `FoxTray.exe` once built, not read from disk). So after replacing artwork, rebuild:

```powershell
./.venv/Scripts/python.exe -m PyInstaller --clean --noconfirm foxtray.spec
```

For the running (dev-mode) tray, just restart the Python process and the cached loader picks up the new PNGs on first `icons.load(state)` call.
